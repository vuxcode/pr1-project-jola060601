6. Problem (Date: 2026-04-07): When I created the interactive keyboard using Javascript, I ran into a few problems. Mostly by appending the keyboard and having the buttons respond. 7. Problem (Date: 2026-04-07): When I created a function to generate a random country from the list, it didn't generate a country name on the console. After searching on the Internet (and a little bit of help using Google AI), I created an additional "script" tag above the "script" tag I'm writing on to load the "js"-file.
8. Problem (Date: 2026-04-09): When I generated lines for each country in my "Hangman Game", no lines showed up. The problem turned out to be a single line of code and how I styled the "<li>" tag on CSS. By removing 'class="currentletter"' from "countryDisplay.innerHTML" the lines showed up. This is because the class "currentletters" should be used when a letter is correctly guessed since the "border-bottom" will become transparent. The "<li>" tag already has lines defined in CSS via its parent-class "currentword".
9. Problem (Date: 2026-04-09): When I cleaned up my code a little bit, something strange happened on the console. It always showed " exists on [insert country here]" when loading the screen. 12. Problem (Date: 2026-04-10): When I wanted to disable a button after it's clicked, I had to make a change in the creation of the keyboard by using "let keyboardButton" because "var keyboardButton" became a global variable stuck on the final keyboard button "-" when calling upon it in the function "startGame()", which meant that no matter which button was clicked, only "-" would be disabled.
