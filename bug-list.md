# Bug List

> Make a list of the things that don't work as expected. Keep a list of things that you have fixed and try to document how you solved them.

1. Problem (Date: 2026-03-30): When I redesigned the webpages (mostly by changing background color and making a menubar at the top) the menu didn't work. Originally I gave the ul element the menu class, but then I put the ul element inside a nav element and gave it the menu class instead, which fixed my problem and gave me a workable menu.
2. Problem (Date: 2026-03-31): When I designed my hangman game, I noticed a problem where the letter ended up being surrounded by a border (each letter being inside an "li" element, all nested inside an "ul" element, both having their own classes). After several edits (and a little bit of help using Google AI), I ended up moving most information to one class, while the second one only needed (for now) the information to make the bottom border transparent after a letter is pressed.
3. Problem (Date: 2026-04-02): When I was designing my hangman game (mostly by adding flexbox properties), I came upon a problem. The h1 and h2 elements were sticking out from the div when I set a height and width to it. After searching on the Internet I changed "height" to "min-height" and the problem was solved.
4. Problem (Date: 2026-04-02): When I changed a test name from "Andorra" to "Saint Vincent and the Grenadines" (the sovereign state with the longest name), I came upon a problem. The name didn't fit the box. After playing around on the developers tool and rechecking WE1 Lesson 10 on Flexbox, I solved the problem.
5. Problem (Date: 2026-04-07): When I merged the two classes "container" and "box" into one class "containerbox", I got some of the word-lines sticking out of the div when minimizing the browser screen. After experimenting with the developers tools, I solved the problem by changing the ID "widenbox" from having a maximum width of 900px to 100% of the screen, fitting everything on a minimized screen and therefore solving the problem.
6. Problem (Date: 2026-04-07): When I created the interactive keyboard using Javascript, I ran into a few problems. Mostly by appending the keyboard and having the buttons respond. I solved these problems by searching on the Internet and using the text from PR1 Lesson 12. Having the buttons respond was a simple change (from var i to let i in the for loop).
7. Problem (Date: 2026-04-07): When I created a function to generate a random country from the list, it didn't generate a country name on the console. After searching on the Internet (and a little bit of help using Google AI), I created an additional "script" tag above the "script" tag I'm writing on to load the "js"-file.
8. Problem (Date: 2026-04-09): When I generated lines for each country in my "Hangman Game", no lines showed up. The problem turned out to be a single line of code and how I styled the "<li>" tag on CSS. By removing 'class="currentletter"' from "countryDisplay.innerHTML" the lines showed up. This is because the class "currentletters" should be used when a letter is correctly guessed since the "border-bottom" will become transparent. The "<li>" tag already has lines defined in CSS via its parent-class "currentword".
9. Problem (Date: 2026-04-09): When I cleaned up my code a little bit, something strange happened on the console. It always showed " exists on [insert country here]" when loading the screen. It turned out that I was calling upon the "startGame" function twice, so I removed the "startGame();" at the bottom of the code and the problem was solved.
10. Problem (Date: 2026-04-09): When I added code so that the counter will increase every time the user guesses wrong, the "Guesses: " part disappeared when I clicked on a button (the counter increased, so that part worked). I solved the problem by simply adding the <span> tag (mentioned in WE1 Lesson 15 today) between "0 / 6" and the problem was solved.
11. Problem (Date: 2026-04-10): When I added code to change the hangman image, I followed the same template as with changing the number of wrong guesses. But it didn't work until I swapped ".innerText" with ".src" for the hangman image to change depending on the number of wrong guesses.
12. Problem (Date: 2026-04-10): When I wanted to disable a button after it's clicked, I had to make a change in the creation of the keyboard by using "let keyboardButton" because "var keyboardButton" became a global variable stuck on the final keyboard button "-" when calling upon it in the function "startGame()", which meant that no matter which button was clicked, only "-" would be disabled. But by using "let" the loop creates an unique variable for each button, allowing the clicked button to be disabled.
13. Problem (Date: 2026-04-14): When I wanted to activate the game modal after finishing the game, it didn't show up when I won under the condition that a country had more than one of the same letter. After searching on the Internet for the solution (including using Google AI), I had to change the win condition, which previously was having the length of the variable currentCountry match the length of the array currentCharacters (which won't happen since the array only lists all UNIQUE letters existing on currentCountry). Instead I created two local variables inside the "startGame" function, the first one got all <li> generated on countryDisplay by currentCountry, the second checking if all <li> are filled in by a letter and then using it in an if-statement to check if it's a win condition. I also had to return "&nbsp;" when generating <li> so that the local variables could also count non-breaking spaces.
14. Problem (Date: 2026-04-14): When I wanted to show the game modal after finishing the game, currentCountry didn't show up when I won. It turned out that when displaying a variable using innerHTML, you have to replace "" with ``, since the former interprets everything as a string while the latter is more flexible.
15. Problem (Date: 2026-04-14): When I restarted the hangman game, the pushed buttons remained disabled and no new lines were generated. I had to include more information to my resetGame function to clear all variables AND UI elements (i.e. text, images, keyboard and modal), including looping through all buttons to remove the "clickedbutton" class.
16. Problem (Date: 2026-04-16): When I moved the script to run the hangman game to a separate "js"-file, both the modal text remained unchanged whether I won or lost and the accented buttons on the keyboard didn't show up. While the former was easy to fix (simply update the lose condition to show different text), the latter required some digging on the Internet before I added <meta charset="UTF-8"> to the <head> element so that the accented letters would show up again.
17. Problem (Date: 2026-04-16): When I added the "show" class to the <div> with the "game-modal" class so that it would show up when running the page, it automatically closed down. After looking at the code and searching on the Internet, it turned out that I had to change the display attibute to show "none" or "flex" depending on whether the "show" is active or not, plus having to remove a line of code so that the "resetGame()" function won't run when loading the page but only when pressing the "Play" button. Best result was to have "display: none" on the "game-modal" class and "display: flex" on the "show" class without touching the "hangman-game-modal" ID.
18. Problem (Date: 2026-04-24): When I generated countries inside randomArray, sometimes there were duplicates. Since the array has arrays inside of it, ".includes()" didn't help. After searching on the Internet and the lesson notes (and a little bit of help using Google AI), it turned out that I had to use ".some()" instead of ".includes()" so that I could check if "code"/"country" already existed inside of randomArray.
19. Problem (Date: 2026-04-27): When I wanted to increase points every time the correct button was pressed, I couldn't link the buttons to a listening event. After a lot of help from my teacher, Colin Myers, plus looking at the lesson notes (and a little bit of help from Google AI), the problem was solved. I had to change the "onclick" function that the buttons call upon, plus I had to add a parameter "clickedButton" so that I could compare the clicked button to the flag.
